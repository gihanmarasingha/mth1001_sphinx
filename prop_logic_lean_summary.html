3.2. Reiteration¶

The reiteration rule states that \(p\) follows from \(h : p\).

This is represented very simply in a term-style Lean proof. The first line of the proof below can be read as ‘Given h : p, here follows a proof of p’.

try it!
example (h : p) : p :=
h -- Application of reiteration.

Reiteration is represented in a tactic-style proof by the exact tactic.

try it!
example (h : p) : p :=
begin
  exact h,
end

3.3. Conjunction (and)¶

3.3.1. Conjunction elimination¶

There are two conjunction elimination rules, left and right.

  • (Left and elimination) given \(h : p \land q\), we have a proof of \(p\).

  • (Right and elimination) given \(h : p \land q\), we have a proof of \(q\).

In a tactic-style proof, we can perform left and right and elimination simultaneously with the cases tactic. Below, cases h with hp hq introduces hypotheses hp : p and hq : q into the context.

try it!
example (h : p ∧ q) : q :=
begin
  cases h with hp hq, -- Equivalent to both left and right and elimination.
  exact hq, -- Closes the goal via reiteration using the proof term `hq`
end

Given h : p ∧ q, a proof term for p ∧ q, the terms h.left and h.right are proof terms for p and q, respectively.

try it!
example (h : p ∧ q) : q :=
h.right -- Term-style right and elimination.

We can use the have, …, from notation to insert a term-style proof into a tactic-style proof. Below, h.right is a proof term for q.

try it!
example (h : p ∧ q) : q :=
begin
  have hq : q, from
    h.right, -- Term-style right and elimination.
  exact hq,
end

3.3.2. Conjunction introduction¶

Forward: given \(h_1 : p\) and \(h_2 : q\), we have a proof of \(p\land q\).

Backward: to prove \(p\land q\), it suffices to prove \(p\) and \(q\).

The split tactic applies conjunction introduction backward.

try it!
example (h₁ : p) (h₂ : q) : p ∧ q :=
begin
  split, -- This replaces the goal p ∧ q with two new goals: 1. p and 2. q.
  { exact h₁, }, -- This closes the goal for p.
  { exact h₂, }, -- This closes the goal for q.
end

The and.intro function, applied to h₁ : p and h₂ : q, gives a proof term for p ∧ q. This is a forward application of conjunction introduction.

try it!
example (h₁ : p) (h₂ : q) : p ∧ q :=
and.intro h₁ h₂

This can also be written using French quotes (a general Lean notation for the so-called constructor of an inductive data type).

try it!
variables p q : Prop

-- BEGIN
example (h₁ : p) (h₂ : q) : p ∧ q :=
⟨h₁, h₂⟩ -- Enter these 'French quotes' with `\<` and  `\>`

This proof term can be used within a tactic block.

try it!
example (h₁ : p) (h₂ : q) : p ∧ q :=
begin
  exact and.intro h₁ h₂ -- Or `exact ⟨h₁, h₂⟩`.
end

3.4. Implication¶

3.4.1. Implication elimination¶

Forward: given \(h_1 : p \to q\) and \(h_2 : p\), we have a proof of \(q\).

Backward: given \(h : p \to q\), to prove \(q\), it suffices to prove \(p\).

The apply tactic uses implication elimination backward.

try it!
example (h : p → q) (k : p) : q :=
begin
  apply h, -- This is a backward proof that changes the goal to proving p.
  exact k,
end

Given h₁ : p → q and h₂ : p, the expression h₁ h₂ is a proof term for q. This is forward implication elimination.

try it!
example (h₁ : p → q) (h₂ : p) : q :=
h₁ h₂ -- h₁ h₂ is the result of implication elimination on h₁ and h₂.

As usual, this proof term can be used within a tactic block using the exact tactic.

try it!
example (h₁ : p → q) (h₂ : p) : q :=
begin
  exact h₁ h₂,
end

3.4.2. Implication introduction¶

Implication introduction: to prove \(p \to q\), it suffices to assume \(h : p\) and derive \(q\).

Tactic-style, if the goal is to prove p → q, then intro h introduces an assumption h : p into the context and replaces the goal with one of proving q.

try it!
example (k : q) : p → q :=
begin
  intro h, -- This is equivalent to 'Assume h : p' in mathematics.
  exact k, -- We close the goal using our proof of q.
end

The term style proof is similar, using assume instead of intro.

try it!
example (k : q) : p → q :=
assume h,
  k

If desired, you can make the type of h explicit, when giving a term-style proof.

try it!
example (k : q) : p → q :=
assume h : p,
  k

3.5. Disjunction (or)¶

3.5.1. Disjunction introduction¶

There are two disjunction introduction rules, left and right.

Forward

  • (Left or introduction) given \(h : p\), we have a proof of \(p \lor q\).

  • (Right or introduction) given \(h : q\), we have a proof of \(p \lor q\).

Backward

  • (Left or introduction) to prove \(p\), it suffices to prove \(p \lor q\).

  • (Right or introduction) to prove \(q\), it suffices to prove \(p \lor q\).

The left and right tactics represent backward left or introduction and right or introduction, respectively.

try it!
example (h : p) : p ∨ q :=
begin
  left, -- This changes the goal, by left or introduction, to proving p
  exact h,
end

Forward, given h : p, the expression or.inl h is a proof term for p ∨ q. Likewise, if h : q, the expression or.inr h is a proof term for p ∨ q.

try it!
example (h : p) : p ∨ q :=
or.inl h

3.5.2. Disjunction elimination¶

Forward: given \(h_1 : p \lor q\), \(h_2 : p \to r\), and \(h_3 : q \to r\), we have a proof of \(r\).

Backward: given \(h : p \lor q\), to prove \(r\), it suffices to (1) assume \(hp : p\) and deduce \(r\) and (2) assume \(hq : q\) and deduce \(r\).

Given h : p ∨ q, the cases tactic applied as cases h with hp hq replaces the goal of proving r with two subgoals: (1) to prove r with an additional assumption hp : p and (2) to prove r with an additional assumption hq : q.

In the example code below, we show two different methods of closing the resulting subgoals, corresponding, in turn, to term-style and tactic-style implication elimination.

try it!
example (h : p ∨ q) (h₂ : p → r) (h₃ : q → r) : r :=
begin
  cases h with hp hq,
  { exact h₂ hp, }, -- `h₂ hp` is implication elimination to give `r`.
  { apply h₃, exact hq, }, -- A tactic-style implication elimination.
end

Here is a more typical example of disjunction elimination.

try it!
example (h₁ : (p ∧ r) ∨ (r ∧ q)) : r :=
begin
  cases h₁ with h₂ h₂,
  { exact h₂.right, }, -- In this subproof, `h₂ : p ∧ r`. The subgoal is `r`.
  { exact h₂.left, }, -- In this subproof, `h₂ : r ∧ q`. The subgoal is `r`.
end

Given h₁ : p ∨ q, h₂ : p → r, h₃ : q → r, the function or.elim applied to h₁, h₂, and h₃ gives a proof-term for``r``.

try it!
example (h₁ : p ∨ q) (h₂ : p → r) (h₃ : q → r) : r :=
or.elim h₁ h₂ h₃

Here is a term-style proof of the previous result.

try it!
example (h₁ : (p ∧ r) ∨ (r ∧ q)) : r :=
or.elim h₁
  (assume h₂ : p ∧ r, h₂.right) -- A term-style proof of `p ∧ r → r`
  (assume h₂ : r ∧ q, h₂.left) -- A term-style proof of `r ∧ q → r`

3.6. If and only if (iff)¶

3.6.1. Iff elimination¶

There are two iff elimination rules, left and right.

  • (Left iff elimination) given \(h : p \leftrightarrow q\), we have a proof of \(p \to q\).

  • (Right iff elimination) given \(h : p \leftrightarrow q\), we have a proof of \(q \to p\).

Note the similarity with this and conjunction elimination.

Given h : p ↔ q, the cases tactic, when applied as cases h with h₁ h₂, decomposes the hypothesis h into two hypotheses, h₁ : p → q and h₂ : q → p. This is the same as left and right iff elimination simultaneously.

try it!
example (h : p ↔ q) : p → q :=
begin
  cases h with h₁ h₂,
  exact h₁,
end

Likewise, given h : p ↔ q, iff.elim_left h is a proof term for p → q and iff.elim_right h is a proof term for q → p.

try it!
example (h : p ↔ q) : p → q :=
iff.elim_left h

3.6.2. Iff introduction¶

Forward: given \(h_1 : p \to q\) and \(h_2 : q \to p\), we have a proof of \(p \leftrightarrow q\).

Backward: to prove \(p \leftrightarrow q\), it suffices to prove \(p \to q\) and \(q \to p\).

The split tactic applies iff introduction backward.

try it!
example (h₁ : p → q) (h₂ : q → p) : p ↔ q :=
begin
  split, -- This replaces the goal `p ↔ q` with 1. p → q and 2. q → p.
  { exact h₁, }, -- Closes the goal `p → q`.
  { exact h₂, }, -- Closes the goal `q → p`.
end

The iff.intro function, applied to h₁ : p → q and h₂ : q → p, gives a proof term for p ∧ q. This is a forward application of iff introduction.

try it!
example (h₁ : p → q) (h₂ : q → p) : p ↔ q :=
iff.intro h₁ h₂

3.7. False and negation¶

3.7.1. False elimination¶

The symbol \(\bot\), referred to as false or contradiction or arbitrary contradiction, is referred to in one fundamental rule of inference, ex falso sequitur quodlibet, also called ex falso or false elimination. This rule states that anything follows from false.

Forward: given \(h : \bot\), we have a proof of \(p\).

Backward: to prove \(p\), it suffices to prove \(\bot\).

The exfalso tactic represents backward false elimination.

try it!
example (h : false) : p :=
begin
  exfalso, -- This changes the goal from `p` to `false`.
  exact h, -- We close the goal with `h`.
end

Given h : false, the expression false.elim h is a proof term for p.

try it!
example (h : false) : p :=
false.elim h

3.7.2. False introduction¶

The expression \(\neg p\) is a shorthand for \(p \to \bot\). The rule of false introduction is thus merely implication elimination in another guise.

Forward: given \(h_1 : \neg p\) and \(h_2 : p\), we have a proof of \(\bot\).

Backward: given \(h : \neg p\), to prove \(\bot\), it suffices to prove \(p\).

The apply tactic uses false introduction backward.

try it!
example (h : ¬p) (k : p) : false :=
begin
  apply h, -- This changes the goal to proving `p`.
  exact k,
end

Given h₁ : ¬p and h₂ : p, the expression h₁ h₂ is a proof term for false. This is forward false introduction.

try it!
example (h₁ : ¬p) (h₂ : p) : false :=
h₁ h₂

3.7.3. Negation introduction¶

As \(\neg p\) is shorthand for \(p \to \bot\), the rule of negation introduction is really just implication introduction.

Negation introduction: to prove \(\neg p\), it suffices to assume \(h : p\) and derive \(\bot\).

Tactic-style, if the goal is to prove ¬p, then intro h introduces an assumption h : p into the context and replaces the goal with one of proving false.

try it!
example (k : false) : ¬p :=
begin
  intro h, -- This is equivalent to 'assume h : p' in mathematics.
  exact k, -- We close the goal using our proof of `false`.
end

The term-style proof is similar.

try it!
example (k : false) : ¬p :=
assume h : p,
  k
